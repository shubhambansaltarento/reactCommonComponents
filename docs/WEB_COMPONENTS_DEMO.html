🎉 Shared Components as Web Components

Framework-agnostic custom elements demo. These work in any framework or vanilla JS!

1. Simple Button Component

<tvs-custom-button id="demo-button" label="Click Me" variant="contained"></tvs-custom-button>

2. Setting Props Programmatically

const btn = document.querySelector('#dynamic-button'); btn.props = { label: 'Dynamic Label', disabled: false };

3. Event Listeners

Events are dispatched via CustomEvent. Listen with addEventListener() for event names.
document.querySelector('#event-button').addEventListener('click', (e) => { console.log('Button clicked!', e.detail); });

4. Component with Multiple Props

<tvs-custom-button label="Complex Button" variant="contained" disabled="false" data-id="123" ></tvs-custom-button>

ℹ️ How Web Components Bridge Frameworks

✅ Why this approach?
  • Use the same component library in React, Vue, Angular, plain JS
  • No framework dependency for consumers
  • Encapsulated behavior via Web Components standard
  • Forward compatible with any JavaScript runtime
📝 Attribute Conversion Rules:
  • on-event-name → dispatches custom event
  • data-x → becomes dataX prop
  • attribute → becomes attribute prop
  • Empty attribute → true
  • JSON strings → parsed to objects/arrays

🚀 Using in Your App

// In your app entry point (main.ts, _app.tsx, index.js, etc) import { registerAllWebComponents } from '@shared/components'; registerAllWebComponents({ prefix: 'tvs' });

After that, all shared components are available as custom elements!
